# 项目分模块术语词典

本文档按模块整理 `XHFrameworkSkill` 项目中的高频术语、类名和字段语义，目标不是讲 Unity 基础，而是帮助你快速把这套项目自己的“语言系统”装进脑子里。

---

## 1. 总体认知

这项目可以先粗分成 4 层：

1. **数据层**：技能图、节点数据、连线、标签、属性配置
2. **编辑器层**：GraphView 技能编辑器，负责配技能图
3. **运行时层**：把技能图解释成可执行逻辑
4. **Demo 接入层**：角色、怪物、UI、Luban 配表、场景演示

最重要的一句翻译：

| 词 | 白话 |
|---|---|
| `Data` | 静态配置 |
| `Spec` | 运行时实例 |
| `Ability` | 技能主控 |
| `Effect` | 数值/状态效果 |
| `Task` | 流程步骤 |
| `Condition` | 条件判断 |
| `Cue` | 表现触发 |
| `ASC` | 单位身上的技能系统中枢 |

## 2. 数据层词典

数据层主要在 `Assets/SkillEditor/Data`。

### 2.1 技能图相关

| 术语 | 白话解释 | 项目里的作用 | 常见位置 |
|---|---|---|---|
| `SkillGraphData` | 一个技能图资源 | 存一整个技能的节点和连线 | `Data/Base/SkillGraphData.cs` |
| `NodeData` | 节点静态数据基类 | 所有节点配置的共同父类 | `Data/Base` |
| `ConnectionData` | 连线数据 | 记录哪个节点的哪个输出口连到下个节点 | `SkillGraphData.connections` |
| `Graph` | 技能图 | 技能不是写死在代码里，而是节点图 | 编辑器 + 运行时 |
| `Node` | 图里的功能块 | 代表技能、伤害、搜索目标、粒子等 | `NodeData` 子类 |
| `Port` | 节点端口 | 节点对外暴露的入口/出口 | GraphView 和运行时执行流 |

### 2.2 节点类型相关

| 术语 | 白话解释 | 说明 |
|---|---|---|
| `NodeType` | 节点类型枚举 | 决定节点属于 `Ability` / `Effect` / `Task` / `Condition` / `Cue` 哪类 |
| `NodeCategory` | 节点分类 | 运行时执行器用它决定如何派发执行 |

`NodeType` 常见枚举值：

- `Ability`：技能根节点，一般是图入口，挂激活、冷却、消耗、动画等端口
- `Animation`：动画时间轴节点，配置动画名、时长、命中时机、Cue 区间
- `Effect` 系：效果节点，负责改属性、挂 Buff、冷却、投射物、位移等
- `Task` 系：任务节点，负责流程动作，比如找目标、结束技能
- `Condition` 系：条件节点，负责判断和分支
- `Cue` 系：表现节点，负责粒子、音效、飘字，不负责核心数值改动

`NodeCategory` 可以理解成运行时对节点做的一次更粗粒度分组，主要分成：

- `Root`
- `Effect`
- `Cue`
- `Task`
- `Condition`

### 2.3 标签相关

| 术语 | 白话解释 | 项目里的作用 |
|---|---|---|
| `GameplayTag` | 单个标签 | 类似 `unitType.hero`、`Buff.DeBuff.Stun` |
| `GameplayTagSet` | 标签集合 | 一组固定标签，常用于配置 |
| `Tag` | 状态/分类标记 | 做条件判断、技能互斥、Buff 状态表达 |
| `Tag` 层级匹配 | 父子标签关系 | `Ability.Attack.Melee` 也算拥有 `Ability.Attack` |
| `GameplayTagLibrary` | 标签常量入口 | 避免全项目手写字符串 |
| `GameplayTagsAsset` | 标签资源 | 标签编辑器/代码生成使用的数据源 |

### 2.4 属性与数值相关

| 术语 | 白话解释 | 项目里的作用 |
|---|---|---|
| `AttrType` | 属性类型 | 比如生命、防御、法防等 |
| `Attribute` | 单个属性对象 | 真正保存基础值、修饰器、当前值 |
| `AttributeSetContainer` | 属性容器 | 单位身上的全部属性集合 |
| `AttributeModifier` | 属性修改器 | 某个效果对某个属性的加减乘除覆盖 |
| `ModifierOperation` | 修改方式 | `Add` / `Multiply` / `Divide` / `Override` |
| `ModifierMagnitudeSourceType` | 修改器数值来源 | 固定值、公式、MMC、SetByCaller |
| `Formula` | 公式 | 按表达式算伤害、治疗、持续值等 |
| `MMC` | 自定义计算模式 | 更结构化的数值计算方式 |
| `Snapshot` | 快照 | 技能生效时把部分属性冻结下来，后续按当时值算 |
| `SetByCaller` | 调用时传值 | 不是静态写死，而是运行时塞进上下文的值 |

### 2.5 持续与堆叠相关

| 术语 | 白话解释 | 项目里的作用 |
|---|---|---|
| `EffectDurationType` | 效果持续类型枚举 | 决定一个效果是瞬时、持续还是永久 |
| `Period` | 周期 | 持续效果每隔多久执行一次 |
| `Stack` | 叠层 | 同类效果叠加层数 |
| `StackType` | 叠层类型 | 按目标堆叠，或按来源堆叠 |
| `StackLimit` | 最大叠层数 | 到上限后可能刷新、拒绝、溢出 |
| `Overflow` | 溢出 | 叠满后新应用时触发的策略 |

`EffectDurationType` 的枚举值：

- `Instant`：瞬时，执行完就结束，如直接扣血
- `Duration`：持续，有持续时间，如眩晕 3 秒
- `Infinite`：永久，理论上不自动过期

## 3. 编辑器层词典

编辑器层主要在 `Assets/SkillEditor/Editor`。

### 3.1 编辑器主结构

| 术语 | 白话解释 | 项目里的作用 | 常见位置 |
|---|---|---|---|
| `SkillEditorWindow` | 技能编辑器主窗口 | 左树、中图、右属性面板 | `Editor/SkillEditorWindow.cs` |
| `SkillGraphView` | GraphView 图面板 | 负责显示节点和连线 | `Editor/Base/GraphView` |
| `NodeInspectorView` | 右侧属性面板 | 编辑当前选中节点的数据 | 编辑器 Inspector |
| `SkillAssetTreeView` | 左侧技能树 | 选择某个技能资源并加载图 | 编辑器左栏 |
| `NodeFactory` | 节点工厂 | 根据 `NodeData` 生成可视化节点 | 编辑器节点创建 |

### 3.2 编辑器常见动作

| 术语 | 白话解释 | 说明 |
|---|---|---|
| `LoadGraph` | 加载图 | 把 `SkillGraphData` 恢复成 GraphView 节点和连线 |
| `SaveGraph` | 保存图 | 把当前 GraphView 写回 `SkillGraphData` |
| `Guid` | 节点唯一标识 | 运行时靠它在缓存里找节点 |
| `OutputPortName` / `InputPortName` | 输出/输入端口名 | 运行时执行流查找的关键 |
| `Serialization` | 序列化 | 节点和连线怎么保存回资源 |

### 3.3 编辑器层你最该记住的话

- 编辑器层只负责“**把图配出来并存成资源**”
- 真正执行技能的是运行时层，不是编辑器层

## 4. 运行时核心层词典

运行时主体在 `Assets/SkillEditor/Runtime`。

### 4.1 系统中枢

| 术语 | 白话解释 | 项目里的作用 | 常见位置 |
|---|---|---|---|
| `AbilitySystemComponent` / `ASC` | 单位技能系统核心组件 | 管属性、标签、技能容器、Effect 容器 | `Runtime/Core/AbilitySystemComponent.cs` |
| `GASHost` | ASC 宿主管理 | 集中注册/管理 ASC | `Runtime/Core/GASHost.cs` |
| `AbilityContainer` | 技能容器 | 存单位拥有的技能，负责激活、取消、Tick | `Runtime/Ability/AbilityContainer.cs` |
| `GameplayEffectContainer` | 效果容器 | 存单位身上的 Buff/CD/持续效果 | `Runtime/Effect/GameplayEffectContainer.cs` |
| `SkillDataCenter` | 技能静态数据缓存中心 | 缓存图、节点、连接，避免反复遍历资源 | `Runtime/Core/SkillDataCenter.cs` |
| `SpecFactory` | Spec 工厂 | 按节点类型创建具体执行实例 | `Runtime/Core/SpecFactory.cs` |
| `SpecExecutor` | 图执行器 | 顺着节点图把技能跑下去 | `Runtime/Core/SpecExecutor.cs` |
| `SpecExecutionContext` | 执行上下文 | 当前这次技能施法共享的数据包 | `Runtime/Core/SpecExecutionContext.cs` |

### 4.2 `Spec` 是什么

`Spec` 是本项目最重要的词之一。

| 术语 | 白话解释 | 常见类 |
|---|---|---|
| `Spec` | 配置在运行时变成的执行实例 | `GameplayAbilitySpec`、`GameplayEffectSpec` |
| `Data -> Spec` | 从静态图纸到活对象 | `SkillGraphData -> GameplayAbilitySpec` |

可以直接这样记：

- `Data` = 图纸
- `Spec` = 图纸跑起来后的实例

| 术语 | 白话解释 | 项目里的作用 |

## 7. Task 模块词典

主要在 `Runtime/Task`。

| 术语 | 白话解释 | 项目里的作用 |
|---|---|---|
| `TaskSpec` | 任务节点基类 | 瞬时执行，不做持续管理 |
| `SearchTargetTaskSpec` | 搜索目标任务 | 用 2D 物理范围检测找目标 |
| `EndAbilityTaskSpec` | 结束技能任务 | 主动终止技能 |
| `SearchShapeType` | 搜索形状枚举 | 决定搜索节点按什么形状做检测 |
| `LineType` | 直线搜索模式枚举 | 决定直线检测方向如何计算 |
| `ParentInputTarget` | 父节点传入目标 | 搜索结果往下游逐个传递时非常关键 |
| `完成效果` / `无目标` / `对每个目标` | 任务输出端口 | 控制搜索后的后续分支 |

`SearchShapeType` 的枚举值：

- `Circle`：圆形范围检测
- `Sector`：扇形范围检测
- `Line`：直线/矩形范围检测

`LineType` 的枚举值：

- `UnitDirection`：按单位朝向延伸
- `BetweenPoints`：按两个位置点之间连线检测
- `AbsoluteAngle`：按绝对角度发射

### 7.1 Task 最容易忽略的点

- `Task` 不等于“协程任务”
- 在这项目里它更像“流程动作节点”
- `SearchTargetTaskSpec` 是把“找谁吃技能”这件事从 Ability 里拆出来

## 8. Condition 模块词典

主要在 `Runtime/Condition`。

| 术语 | 白话解释 | 项目里的作用 |
|---|---|---|
| `ConditionSpec` | 条件节点基类 | 做判断，不直接改状态 |
| `AttributeCompareConditionSpec` | 属性比较条件 | 比较生命、法力等数值 |
| `CompareOperator` | 比较运算符枚举 | 决定条件节点怎么比较两个值 |
| `AttributeValueType` | 比较值类型枚举 | 决定拿固定值还是百分比做比较 |
| `是` / `否` | 条件输出口 | 判断结果分支 |

`CompareOperator` 常见值：

- `Equal` / `NotEqual`
- `Greater` / `GreaterOrEqual`
- `Less` / `LessOrEqual`

`AttributeValueType` 常见值：

- `Fixed`：固定值
- `Percentage`：百分比

一句话：

`Condition` 在这项目里就是“图里的 if 节点”。

### 9.2 Cue 和 Effect 的差别

| 对比项 | `Effect` | `Cue` |
|---|---|---|
| 主要职责 | 改数值、改状态 | 做表现 |
| 典型结果 | 扣血、上 Buff、上 CD | 粒子、音效、飘字 |
| 是否影响战斗结算 | 通常会 | 通常不会 |

## 13. 常见易混词对照

### 13.1 `Data` vs `Spec`

| 词 | 含义 |
|---|---|
| `Data` | 配置，放资源里，不自己执行 |
| `Spec` | 运行时实例，真的会执行逻辑 |

### 13.2 `Ability` vs `Effect`

| 词 | 含义 |
|---|---|
| `Ability` | 一次技能施放的总控 |
| `Effect` | 技能里某个具体效果，如伤害、治疗、Buff |

### 13.3 `Effect` vs `Cue`

| 词 | 含义 |
|---|---|
| `Effect` | 改游戏状态 |
| `Cue` | 播放视觉/听觉表现 |

### 13.4 `Task` vs `Condition`

| 词 | 含义 |
|---|---|
| `Task` | 做动作、跑流程 |
| `Condition` | 做判断、分支 |

### 13.5 `MainTarget` vs `Targets` vs `ParentInputTarget`

| 词 | 含义 |
|---|---|
| `MainTarget` | 当前技能的主要目标 |
| `Targets` | 当前上下文里的目标列表 |
| `ParentInputTarget` | 上一个节点逐个传下来的当前目标 |

### 13.6 `AssetTags` vs `GrantedTags`

| 词 | 含义 |
|---|---|
| `AssetTags` | 技能/效果自身的分类标签 |
| `GrantedTags` | 效果生效时临时附加到目标身上的标签 |

### 13.7 `Remove` vs `Cancel` vs `Expire`

| 词 | 含义 |
|---|---|
| `Remove` | 主动移除效果并做清理 |
| `Cancel` | 提前打断 |
| `Expire` | 到时间自然过期 |
